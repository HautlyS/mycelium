Welcome to Mycelium

The first decentralized, self-replicating, self-improving AI network. From each according to their compute, to each according to their need.

What is Mycelium?

Mycelium is a decentralized AI system that runs across heterogeneous nodes (native GPU + browser WebGPU via WASM). It uses latent-space continuous representations instead of tokenized generation, self-replicates across devices like biological spores, and self-improves via federated LoRA tuning.

🌱

Self-Replication

Nodes create and share "spores" containing compressed model weights, LoRA adapters, and runtime. The network grows organically like fungal spores.

🧠

Self-Improvement

Each node continuously learns from interactions. Gradient deltas are shared via P2P, aggregated through federated averaging, improving the global model.

⚡

Distributed Compute

Model layers are split across nodes dynamically. MoE routing enables natural parallelism with 64 experts, only 4 active per token.

🔐

Privacy First

User data never leaves your node. Only gradient deltas (with noise) are shared. Cryptographic verification prevents unauthorized access.

Core Concepts

1. Latent-Space Processing

Instead of: input → tokenize → transformer → detokenize → output

We do: input → encode → latent vector → transform in latent space → decode

Benefits:

  • Continuous representation enables interpolation, morphing, blending
  • Compresses ~4-8x vs token sequences (230B MoE hidden_dim=6144)
  • Latent ops are matmuls — perfect for distributed tensor parallel
  • Enables "thought vectors" that flow between nodes without tokenization

2. Self-Replication (Spore Protocol)

A "spore" is a minimal self-contained package:

  • genome: compressed model weights (GGUF shard)
  • instincts: initial LoRA adapter (learned behaviors)
  • seed: WASM binary of the node runtime
  • target: addressing hints for propagation

Scaling Analysis

"Nature scales through decentralization. So do we."

Scaling Philosophy

Mycelium follows Modified Metcalfe's Law:

Network Value ≈ O(n × log(n))

Why not O(n²)? Because not all nodes need direct connections. The DHT + gossipsub architecture provides logarithmic discovery and propagation.

Security Model

"Trust nothing. Verify everything. Assume some nodes are malicious."

Security Philosophy

  1. Zero Trust — No node is trusted by default. All messages are verified.
  2. Defense in Depth — Multiple layers of security.
  3. Graceful Degradation — Security failures affect individual nodes, not the network.
  4. Privacy by Design — User data never leaves their node.

Spore Verification Chain

1. CRC32 Check → Detects transport corruption (fast)
     ↓ PASS
2. SHA-256 Hash → Verifies weight integrity (cryptographic)
     ↓ PASS
3. Ed25519 Signature → Authenticates author (non-repudiation)
     ↓ PASS
4. Weight Registry → Verifies against pinned hash
     ↓ PASS
5. Load into candle → Only now is it safe

  2. Make changes and implement your feature
  3. Run tests: cargo test --workspace
  4. Run clippy: cargo clippy --workspace -- -D warnings
  5. Format code: cargo fmt --workspace
  6. Commit with descriptive message
  7. Push and create pull request

Coding Standards

  • Follow Rust API Guidelines
  • All public APIs must have rustdoc comments
  • Use thiserror for error types
  • Use tracing for structured logging
  • Write tests for all new functionality

Pull Request Process

2. Open an issue first (for significant changes)
3. Write tests for new functionality
4. Update documentation
5. Submit PR with clear description
6. Address reviewer feedback
7. Merge after approval

Philosophy & Ethics

"From each according to their compute, to each according to their need."

Core Philosophy

Decentralization as Liberation

Centralization of powerful technology inevitably leads to concentration of power. Decentralization is not just a technical choice — it's a political and ethical one.

Collective Intelligence

Intelligence emerges from connections, not isolation. A network of modest intelligences, properly connected, can surpass any individual intelligence.

Gift Economy

The most resilient systems are based on giving, not extracting. Nodes contribute compute, storage, and bandwidth freely. In return, they receive access to collective intelligence.

Ethical Principles

Principle Implementation
Privacy FirstUser data never leaves node, only gradient deltas shared
TransparencyAll code, weights, and updates are open and auditable
AccessibilityRuns on browsers, phones, laptops — no gatekeeping
AutonomyUsers control their own AI experience
BeneficenceNetwork benefits all participants, no extraction

The Problem with Centralized AI

  • Power concentration — Few corporations control frontier AI
  • Epistemic monoculture — Limited diversity of perspectives
  • Data exploitation — User data extracted without consent
  • Compute waste — Redundant datacenters consuming energy

The Mycelium Alternative

  • Distributed power — Community governance, not corporate control
  • Cognitive diversity — Different nodes, different models, diverse perspectives
  • Resource efficiency — Leverages idle compute, no redundant datacenters
  • Resilience — Survives corporate bankruptcy, censorship, disasters
